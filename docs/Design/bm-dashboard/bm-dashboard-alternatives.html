Branch Manager Dashboard - 5 Redesign Alternatives

Critique-driven redesigns following SG Design System | Based on UX best practices for decision-making dashboards

Current Dashboard - Critique
Analysis

The current BM dashboard provides comprehensive data but suffers from information overload with 7 distinct sections competing for attention. Critical information like the real-time queue is buried below the fold, and related decision-making data is scattered across multiple locations.

Strengths

  • Comprehensive data coverage - all operational data in one view
  • Real-time updates via Socket.io
  • Good trend indicators (vs yesterday comparisons)
  • Color-coded status indicators on counters
  • Branch ranking creates healthy competition

Weaknesses

  • 7 sections competing for attention - no clear hierarchy
  • Violates F-pattern - critical queue buried below fold
  • Alerts banner too subtle (inline text, easy to miss)
  • Counter controls separated from queue visualization
  • All cards use identical styling - no size differentiation
  • Mobile/tablet unfriendly - 5-column scorecard collapses poorly
  • Must mentally synthesize data from multiple sections
  • No "at a glance" answer to "Are we OK?"
Key insight: To answer "Do I need more counters?", the manager must check: waiting count (top) + open counters (Quick Actions) + queue preview (scroll down) + SLA (right side). Related information is spread across 4 locations.
Alternative 1: Command Center
Operations Focus

Single-screen operational command center with clear zones. Hero status panel shows 4 mega-metrics with traffic light colors. Counter controls are placed directly adjacent to queue visualization so managers can see cause and effect together. Secondary information (performance, ranking) is collapsed into an accordion.

Hero status panel with 4 mega-metrics
Traffic light colors (green/amber/red)
Combined counters + queue view
Progressive disclosure via accordion

Pros

  • Immediate status comprehension from hero panel
  • Related data grouped together (counters + queue)
  • Fits on one screen without scrolling
  • Alerts integrated into header (always visible)

Cons

  • Less detail visible by default
  • Requires clicks for performance data
  • Accordion may be forgotten

Alternative 2: Split-Screen Operations
Control vs Monitor

Left = Control, Right = Monitor. Clear action vs. observation split with a fixed sidebar for all controls. Kanban-style queue view (Waiting -> Called -> Serving) shows ticket flow visually. Horizontal mini-leaderboard conserves space while still showing team performance.

Fixed left sidebar for controls
Kanban queue visualization
Natural left-to-right workflow
Alerts at bottom right (prominent but not blocking)

Pros

  • Clear separation of concerns
  • Natural left-to-right workflow
  • Queue visualization more intuitive
  • Controls always visible

Cons

  • Narrower control panel on small screens
  • Less space for detailed analytics
  • May feel cramped on tablets

Alternative 3: Priority-First Dashboard
Attention-Driven

Show what needs attention first. Everything else is secondary. Attention section at top with actionable alerts that include inline actions (bump VIP, open counter). Status strip provides key metrics in a single row. Branch ranking moves to a separate tab to reduce clutter.

Attention section at top
Inline actions on alerts
Status strip for key metrics
Three-column operations

Pros

  • Immediate focus on problems
  • Actions embedded in context
  • Cleaner visual hierarchy
  • Reduces cognitive load

Cons

  • Empty "Attention" section when all is well
  • Less competitive context (no ranking)
  • May need empty state design

Alternative 4: Dashboard Tabs
Role-Based Views

Role-based views with different priorities at different times. Operations tab focuses on real-time queue, Performance tab for team management, Analytics tab for trends and competition. Each tab can be fully expanded without clutter from other sections.

Tabbed interface separates concerns
Operations tab = real-time
Performance tab = team
Analytics tab = trends

Pros

  • Cleaner focused views
  • Less overwhelming
  • Can expand each tab fully
  • Good for different tasks

Cons

  • Requires navigation
  • May miss alerts on other tabs
  • More clicks for full picture

Alternative 5: Metric-Driven Hierarchy
Recommended

Size = Importance. Visual weight guides attention. Mega hero metrics (text-6xl for waiting count) provide instant status comprehension. Numbered queue list shows clearer priority than grid. Collapsible secondary section for team/rank/daily keeps interface clean while information remains accessible.

Mega hero metrics (size = importance)
Numbered queue list (clear priority)
Inline alert icons on long-wait tickets
Collapsible secondary section

Pros

  • Instant comprehension of status
  • Clear visual hierarchy
  • Most important data largest
  • Follows F-pattern reading
  • Mobile-friendly - single column hero scales well

Cons

  • Less information density
  • Team performance de-emphasized
  • Requires expand for full details

recommend Recommendation: Alternative 5 - Metric-Driven Hierarchy

Alternative 5 is recommended because it addresses the core weaknesses of the current dashboard while maintaining the SG design aesthetic:

  • Follows F-pattern - Critical metrics (waiting count, wait time) positioned top-left with large visual weight
  • Progressive disclosure - Secondary data (team, ranking, daily target) in collapsible section
  • Immediate status - One glance tells health (green/amber/red on mega metrics)
  • Actionable grouping - Counters placed directly adjacent to queue list
  • Adheres to design system - Uses SG Red sparingly, monochrome foundation, Material Symbols
  • Mobile-friendly - Single-column hero scales well, numbered list works on all screens
